🏫 RESERVASALA - SISTEMA DE RESERVA DE SALAS CBIOT
=====================================================

Este é um sistema completo de reserva de salas com:
✓ Backend em Python (Flask + SQLAlchemy)
✓ Banco de dados PostgreSQL
✓ Frontend em React (Vite) com interface responsiva
✓ API REST com documentação implícita
✓ Painel administrativo protegido por senha

🚀 PRÓXIMOS PASSOS:

1. INSTALE O POSTGRESQL (se ainda não tiver)
   - Windows: https://www.postgresql.org/download/windows/
   - Mac: brew install postgresql
   - Linux: sudo apt install postgresql

2. CRIE O BANCO DE DADOS
   Abra o terminal do PostgreSQL (psql ou pgAdmin) e execute:
   CREATE DATABASE reservasala;

3. CONFIGURE O BACKEND
   - Copie backend/.env.example para backend/.env
   - Edite o .env com as credenciais do seu PostgreSQL:
     DATABASE_URL=postgresql://usuario:senha@localhost:5432/reservasala
     ADMIN_PASSWORD=admin123   # senha do painel admin
     SECRET_KEY=uma-chave-secreta-qualquer

4. INICIE O BACKEND (Flask)
   cd backend
   python -m venv venv
   source venv/bin/activate   # (Linux/Mac) ou .\venv\Scripts\activate (Windows)
   pip install -r requirements.txt
   python app.py
   O servidor Flask estará rodando em http://localhost:5000.

5. INICIE O FRONTEND (React)
   Abra outro terminal:
   cd frontend
   npm install
   npm run dev
   O frontend estará disponível em http://localhost:5173.

6. ACESSE NO NAVEGADOR
   - Página pública (reservas): http://localhost:5173
   - Área administrativa: http://localhost:5173/admin (senha definida no .env, padrão admin123)

💡 RECURSOS DA APLICAÇÃO

✓ Fazer reservas com seleção de sala, data e horário (08:00–19:00, horários cheios)
✓ Consultar disponibilidade em tempo real (blocos de 1 hora coloridos)
✓ Todas as reservas aparecem na lista de "Reservas Confirmadas"
✓ Login administrativo com sessão protegida
✓ Gerenciar salas (criar, excluir) – apenas admin
✓ Cancelar qualquer reserva – apenas admin
✓ Notificações (toast) ao criar reserva, excluir sala ou cancelar reserva
✓ Interface totalmente responsiva (mobile-friendly)

📊 BANCO DE DADOS

As tabelas salas e reservas são criadas automaticamente na primeira execução do backend (db.create_all()).
Você pode gerenciar salas pelo próprio painel administrativo ou adicionar registros diretamente no banco.

🔌 API REST (endpoints disponíveis)

Método | Endpoint                               | Descrição
GET    | /api/salas                             | Lista todas as salas
POST   | /api/salas (admin)                     | Cria uma nova sala
DELETE | /api/salas/<id> (admin)                | Remove uma sala
GET    | /api/reservas                          | Lista reservas (com filtros opcionais)
POST   | /api/reservas                          | Cria uma nova reserva
DELETE | /api/reservas/<id> (admin)             | Cancela uma reserva
GET    | /api/disponibilidade?sala_id=...&data=... | Retorna blocos de horários livres/ocupados
